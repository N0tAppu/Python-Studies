Bubble sort algorithm.
---------------------

It is a simple algorithm used to sort data in a list.

for example.

num = [2,8,4,6,1]
swapped = True
while swapped:
   swapped = False
   for i in range (len(num)-1):
       if num[i] > num[i+1]:
           swapped = True
           num[i],num[i+1] = num[i+1],num[i]
print(num)

This code will sort the list in ascending order.

Note!!!: The swapped variable is IMPORTANT for the functioning of the algorithm here.

**Important Info about lists**
===============================
list_1 = [1]
list_2 = list_1
list_1[0] = [2]
print(list_2)


The output will be [2].

This is because: The assignment: list_2 = list_1 copies the name of the array, not its contents.
In effect, the two names (list_1 and list_2) identify the same location in the computer memory.
Modifying one of them affects the other, and vice versa.

You could say that:
*The name of an ordinary variable is the name of its content;
*The name of a list is the name of a memory location where the list is stored.

So to prevent this and instead of copying the name of a list, we can make it so 
a list will only store data of the element we want to that list.
for this we use "Slice"

eg:
 
num1 = [1]
num2 = num1[:]
num1[0] = 2
print(num2)

output:
1

[:] is called a slice. it is used at the back of the name of the list we equated to prevent the 
updating of data for the 2nd lsit.

*A slice is able to create a brand new list instead of sharing same data as equated list.


It can also be used to choose elements inside a list.

num =[1,2,3,4,5]
list = num[1:4]
print(list)

Output [1,2,3]

So the format of slicing is actually like this

list = num[start:end]

So the start value is the value which is the starting of element in the new list.
While the end value is the starting of the number which is not added to the list.

So basically the list will have (end - start) elements.

Tricks with this slicing:

We could leave this each side of the slice empty to choose entire list.
But but emptying either one side is a different story.

Look at this:

num = [1,2,3,4,5,6]
list = num[:5]
print(list)

Output = [1,2,3,4,5]

*When we leave the start value empty, it thinks that the start value is from 0 (the first element)
*When we leave the end value empty instead will make it that the end value = len(my_list)

Negatives in list slicing:
-------------------------
num = [1,2,3,4,5,6]
list = num[1:-1]
print(list)

Output = [2,3,4,5]

This is because we know in indexing of list elements, negative number for indexing means numbering from the back. 
Thats why this happens.

Looks at this;

num = [1,2,3,4,5,6]
list = num[-1:1]
print(list)

Output: Error

Here the start value is negative. Its not possible. naming of list always starts from 0.

Functions
==========
It often happens that a particular piece of code is repeated many times in your program. It's repeated either literally, or with only a few minor modifications,
 consisting of the use of other variables in the same algorithm. It also happens that a programmer cannot resist simplifying the work, and begins to clone such
 pieces of code using the clipboard and copy-paste operations.

It could end up as greatly frustrating when suddenly it turns out that there was an error in the cloned code. 
The programmer will have a lot of drudgery to find all the places that need corrections. 
There's also a high risk of the corrections causing errors.

We can now define the first condition which can help you decide when to start writing your own functions:
 if a particular fragment of the code begins to appear in more than one place, consider the possibility of 
isolating it in the form of a function invoked from the points where the original code was placed before.
